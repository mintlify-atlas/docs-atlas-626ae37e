## Prerequisites

Before you begin, make sure you have the following installed:

- **Node.js 14.x** — The API requires Node.js 14 (see `engines` in `package.json`)
- **npm** — Comes bundled with Node.js
- **PostgreSQL** — A running PostgreSQL instance with a `Prueba` table

<Info>
  The database connection uses SSL (`ssl: true`). Make sure your PostgreSQL instance supports SSL connections, or see the [database guide](/guides/database) for local development options.
</Info>

## Setup steps

<Steps>
  <Step title="Clone the repository">
    Clone the Eventos API repository to your local machine.

    ```bash
    git clone https://github.com/Shotylan/Eventos.git
    cd Eventos
    ```
  </Step>

  <Step title="Install dependencies">
    Install all required npm packages.

    <CodeGroup>
      ```bash npm
      npm install
      ```
      ```bash yarn
      yarn install
      ```
    </CodeGroup>

    This installs: `express`, `cors`, `dotenv`, and `pg`.
  </Step>

  <Step title="Configure environment variables">
    Create a `.env` file in the project root with your database connection string.

    ```bash .env
    DATABASE_URL=postgresql://user:password@localhost:5432/eventos
    PORT=3000
    ```

    <Warning>
      Never commit your `.env` file to version control. Add it to `.gitignore` to keep your credentials safe.
    </Warning>

    Replace `user`, `password`, `localhost`, `5432`, and `eventos` with your actual PostgreSQL credentials and database name. The `PORT` variable is optional and defaults to `3000`.

    See the [environment setup guide](/guides/environment-setup) for full details on each variable.
  </Step>

  <Step title="Start the server">
    Run the API server using the start script.

    ```bash
    npm start
    ```

    You should see the server start without errors. By default it listens on port `3000`.

    <Tip>
      If port 3000 is already in use, set a different port in your `.env` file: `PORT=3001`.
    </Tip>
  </Step>

  <Step title="Test the endpoint">
    With the server running, call the `/lista` endpoint to retrieve participants.

    ```bash
    curl http://localhost:3000/lista
    ```

    A successful response returns a JSON array of participant records:

    ```json
    [
      {
        "id": 1,
        "nombre": "Ana García",
        "correo": "ana.garcia@utmach.edu.ec",
        "evento": "Congreso de Ingeniería 2024"
      },
      {
        "id": 2,
        "nombre": "Luis Pérez",
        "correo": "luis.perez@utmach.edu.ec",
        "evento": "Congreso de Ingeniería 2024"
      }
    ]
    ```

    If the `Prueba` table is empty, the response will be an empty array `[]`.
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Error: Cannot connect to the database">
    Verify that your `DATABASE_URL` in `.env` is correct and that your PostgreSQL instance is running. Make sure the database and `Prueba` table exist. See the [database guide](/guides/database) for the recommended table schema.
  </Accordion>

  <Accordion title="CORS error in the browser">
    The API only allows requests from `https://eventosutmach.000webhostapp.com` in a browser context. Direct `curl` calls are not subject to CORS. See the [authentication guide](/guides/authentication) for details on how CORS is configured.
  </Accordion>

  <Accordion title="SSL connection error">
    The `pg.Pool` is configured with `ssl: true`. If your local PostgreSQL instance does not have SSL enabled, you may need to adjust the pool configuration for local development. See the [database guide](/guides/database) for guidance.
  </Accordion>

  <Accordion title="Wrong Node.js version">
    The API requires Node.js 14.x. Run `node --version` to check your version. Use a version manager like `nvm` to switch: `nvm use 14`.
  </Accordion>
</AccordionGroup>
